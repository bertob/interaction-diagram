Environment

Physical Affordance

Physical action the user performs, e.g. touching, scrolling, right-clicking

System Action

System response to the physical action, e.g. a menu opens, the zoom level changes, a list item is added

Action Presentation

How affordance and system action are presented to the user, e.g. a scrollbar indicates scrollability, a blinking cursor the possibility of text input

No Action Presentation

Availability of the action is not made explicit in the interface, e.g. keyboard shortcuts

Before

Preview

"What is going to happen as a result of this action?"

Passive

Static elements previewing the outcome of the action, e.g. text labels, icons, graphical elements, continuous animations

Active

User-triggered elements previewing the outcome of the action, e.g. Alt text, hovercards, user-triggered animations

No Preview

No way to know what will happen without triggering the action, e.g. clicking an icon with unclear meaning and no description

During

Feedback

"What is the current progress of the action?"

User-Controlled

Continous transition between initial and final state, the user controls time during the action

Direct Manipulation

Direct mapping of movement to result, e.g. drag and drop, scrolling, pinch-to-zoom

Indirect Manipulation

Indirect mapping of movement to other variables, e.g. color picker slider, pull down to minimize

Autonomous

Continous transition between initial and end state, time is automatic

Interactive

User can stop, pause or change the course of the action once it has started, e.g. file transfer with pause and cancel

Non-Interactive

Once started, the action cannot be stopped or changed (static animation)

No Feedback

No transition between initial and final state (hard cut)

Partly Opaque

Only some parts of the action are invisible or uncontrollable, e.g. hard cut reload after pull to refresh

After

Reversibility

"How do I go back?"

Intuitive

Opposite of the action will return system to its previous state, e.g. scrolling back, moving a slider to its previous value

Non-Intuitive

Different action enables returning to the previous state, e.g. back/undo button

No Reversibility

Not reversible without considerable effort or additional knowledge, e.g. deleting a file after creating it
